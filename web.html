Part One: Solidify Terminology

  1. What is HTTP:
    HTTP is a protocol, a set of rules for the communication between the browser and the web server.
  2. What is a URL:
    Universal resource locators it's basically a set of instructions how to find some resources and to retrieve them.
  3. What is DNS:
    Domain Name System takes the hostname and turn it into an IP address and vice-versa.
  4. What is a query string:
    a query string is a set of words that we enter into the URL in order for the browser to retrieve those data from the database.
  5. What are two HTTP verbs and how are they different:
    1. Get = to get some data from the server, 2. Post = add some data to the server.
  6. What is an HTTP request:
    an HTTP request is request from a client (the browser) to the server, that follows the HTTP protocol.
  7. What is an HTTP response:
    an HTTP response is the response that the server returns to the client.
  8. What is an HTTP header? Give a couple examples of request and response headers you have seen.
    An HTTP header is a field that will contain information about the client's request, if it worked
    We've seen Request headers(method, the URL requested,accept-language) and Response headers(content language and type, status code, response body)
  9. What are the processes that happen when you type “http://somesite.com/some/page.html” into a browser

    1. Turn somesite.com into IP
    2. Connect to IP
    3. On port 80 (the default)
    4. Using the HTTP protocol
    5. Ask for /some/page.html
